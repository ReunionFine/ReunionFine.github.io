对git的认识（3）

git

在使用git时的主要问题解决

  • 版本回退（未提交到远程库）

    1. 先用git log –pretty=oneline/git reflog查看之前提交的id
    2. 使用git reset –hard id

  • （未提交到远程库）

    1. 丢弃工作区最近一次的修改没提交到版本库 git checkout – 文件名（用版本库替代工作区的文件内容）
    2. 丢弃最近一次已经提交到版本库的修改 git reset HEAD 文件名

  • 解决冲突，当不同的人修改了文件的同一个地方会出现此问题

    1. 用cat 查看文件内容（查看不同）
    2. 手动修改提交

  • 解决bug

    1. 先冻结没有提交的工作

    2. 选择在哪条分支上修改

    3. 创建一个新分支（专门解决问题）

    4. 修复问题（提交时看下id）

    5. 切换分支完成合并，删除解决问题的分支

    6. 将修复bug的提交复制到所有分支 git cherry-pick id

  • 多人协作时遇到问题(你和同伴推送的同一个分支有冲突)

    1. git pull（先将同伴最新的提交从远程库抓下来，但有可能会显示本地分支没有和远程同伴的分支链接此时用git branch –set-upstream-to=/分支名 分支名先链接再git pull)
    2. 手动解决冲突
    3. 推送到远程库

对git的认识（2）

git

接着follow up啦（主要列举一些常见的我也希望自己一定要记住的）

  1. git add 文件名 （加到暂存区）
  2. git commit -m “ “ （提交到版本库）
  3. cat 文件名 （看工作区当前的内容）
  4. git status
  5. git diff
  6. git log (看历史记录后可再加指令更详细)
  7. git rest –hard id (用于版本回退)
  8. git rm 文件名 （在工作区中删除文件）
  9. git rm+git commit (在版本库中删掉)
  10. git push -u origin 分支名 （将本地分支和远程库链接并且推送地分支到远程库）
  11. git push origin 分支名 （ 将本地分支的修改推送到远程库）
  12. git switch -c 分支名 （建立新的分支）
  13. git branck (查看当前所有的分支)
  14. git branck -d 分支名 （删除分支）/ git branck -D 分支名 （强制删除分支）
  15. get switch 分支名(切换到此分支)
  16. git merge 分支名 （合并当前分支和此分支）/ git merge –no-ff-m “” 分支名 （和前者作用相同不过不会删除掉分支的历史记录）
  17. git branch -d 分支名 （删除此分支）
  18. git stash (在修改未提交时冻结使得工作区是干净的)
  19. git stash list (查看冻结前的工作区)
  20. git stash pop (恢复冻结的工作区内容，使工作区回到未冻结前的状态)
  21. git cherry -pick id (将在某个分支上提交的修改复制到此分支上)
  22. git remote -v (查看远程库的信息)
  23. git pull (该命令将获取远程服务器上的变更，并合并到你的工作目录，针对于你和同伴对同一个分支进行修改，合并)
  24. git rebase (使历史记录即git log为一条直线)

对git的认识（1）

git

对git的认识有四部分follow up

  • 将ssh中的文件后缀为.pub的粘到github中的固定页面

    1. 先建立远程库将远程库和本地库连接起来，打开GitHub他的空白库下有方法介绍（一般用克隆，注意建立远程库时一定要勾选add README,按提示方法克隆完毕后可以用命令cd 库名+ls查看）

    2. 先建立本地库，将本地库和建立好的远程库链接起来（先手动创建一个文件夹然后用git bush打开输入命令 git init 把它变成git可管理的仓库然后在里面添加文件，本地版本库就建好了，随后建立一个远程库，根据github下面的提示在我们的本地库下面打开git bush输入命令即可链接）

      （在hexo的安装里呢采用的是先建本地库再建远程库然后在hexo这个本地库的config.yml文件中的deploy属性下添加type:git repo值 branch值就ok啦）
