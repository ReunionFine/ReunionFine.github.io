对git的认识（2）

git

接着follow up啦（主要列举一些常见的我也希望自己一定要记住的）

  1. git add 文件名 （加到暂存区）
  2. git commit -m “ “ （提交到版本库）
  3. cat 文件名 （看工作区当前的内容）
  4. git status
  5. git diff
  6. git log (看历史记录后可再加指令更详细)
  7. git rest –hard id (用于版本回退)
  8. git rm 文件名 （在工作区中删除文件）
  9. git rm+git commit (在版本库中删掉)
  10. git push -u origin 分支名 （将本地分支和远程库链接并且推送地分支到远程库）
  11. git push origin 分支名 （ 将本地分支的修改推送到远程库）
  12. git switch -c 分支名 （建立新的分支）
  13. git branck (查看当前所有的分支)
  14. git branck -d 分支名 （删除分支）/ git branck -D 分支名 （强制删除分支）
  15. get switch 分支名(切换到此分支)
  16. git merge 分支名 （合并当前分支和此分支）/ git merge –no-ff-m “” 分支名 （和前者作用相同不过不会删除掉分支的历史记录）
  17. git branch -d 分支名 （删除此分支）
  18. git stash (在修改未提交时冻结使得工作区是干净的)
  19. git stash list (查看冻结前的工作区)
  20. git stash pop (恢复冻结的工作区内容，使工作区回到未冻结前的状态)
  21. git cherry -pick id (将在某个分支上提交的修改复制到此分支上)
  22. git remote -v (查看远程库的信息)
  23. git pull (该命令将获取远程服务器上的变更，并合并到你的工作目录，针对于你和同伴对同一个分支进行修改，合并)
  24. git rebase (使历史记录即git log为一条直线)
